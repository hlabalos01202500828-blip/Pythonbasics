📖 Python Basics Overview

BES103 Lesson 3 covers the foundational concepts of Python programming. Master these before Quiz 3 (April 28) which covers Algorithms and Python Basics.

🖨️ print() Statement

Display text or values. Use print() for blank lines. Supports escape sequences and both quote styles.

# Comments

Start every comment line with #. Comments can also appear at the end of a code line.

📦 Variables

Just names — no declaration needed! Created on first assignment. No intrinsic data type (values have types).

🔢 Data Types

Python has int, float, complex, and str. Values are immutable number objects. Python auto-detects types.

➕ Arithmetic

Standard +, -, *, /. Use ** for exponentiation. + concatenates strings, * repeats them.

⬜ Whitespace

Indentation defines code blocks — not curly braces. Must be consistent (tabs OR spaces, not both).

🔗 Assignment

Creates references, not copies. Multi-assign with x, y = 1, 2. Swap with x, y = y, x.

📛 Naming

Case-sensitive. Cannot start with a number. Use joined_lower for functions, StudlyCaps for classes.

Quick Reference Code Sample

# Variables and Arithmetic x = 34 - 23 # integer: 11 y = "Hello" # string z = 3.45 # float if z == 3.45 or y == "Hello": x = x + 1 y = y + " World" # string concat print(x) # 12 print(y) # Hello World

🖨️ Print Statement & Comments

The print() Statement

  • print("text") — outputs text to console
  • print() — prints a blank line
  • Escape sequences: \" includes a quote inside string
  • Strings can use single ' OR double " quotes
  • "abc" == 'abc' — both are identical

Comments

  • Start with # — the "pound sign"
  • Each commented line must start with #
  • Can appear at end of a line (inline comment)
  • Python ignores everything after #
  • Good practice: comment author, date, purpose at top

Code Examples

# swallows.py — demonstrates print() usage print("Hello, world!") print() # blank line print("Suppose two swallows \"carry\" it together.") print('African or "European" swallows?') Output: Hello, world! Suppose two swallows "carry" it together. African or "European" swallows?

📦 Variables

Key Facts

  • Variables are names for values
  • Created by use — no declaration necessary
  • Variables can be assigned to other variables
  • In Python, variables are just names
  • Variables do not have data types — values do
  • You MUST assign a value before using a variable
  • Python does NOT initialize variables with a default
  • Python determines type from the assigned value

Error: Using Undefined Variable

>>> planet = 'Sedna' >>> print(plant) # typo: plant vs planet Traceback (most recent call last): File "<pyshell>", line 1, in <module> print plant NameError: name 'plant' is not defined # Python does NOT initialize to a default value!

Variable Assignment Examples

>>> planet = 'Pluto' # planet → 'Pluto' >>> print(planet) Pluto >>> moon = 'Charon' # moon → 'Charon' >>> planet = 9 # planet now points to 9 # 'Pluto' is garbage collected 🗑️

Type is in the Value, Not the Name

The same variable name can hold different types at different times because values have types, not variable names. Python figures out the type automatically based on what is assigned.

planet = 'Pluto' # value is str planet = 9 # value is int — same name, new type! planet = 3.14 # value is float

🔢 Types, Conversion & Arithmetic

Python's 4 Numeric Types

  • int — signed integers: x = 5
  • long — large integers (can be octal/hex): x = 999999999L
  • float — floating point: x = 3.456
  • complex — complex numbers: x = 3+4j

Number types are immutable — changing the value creates a new object, not modifying the existing one.

Basic Datatypes Summary

# int (default for whole numbers) z = int(5/2) # Answer: 2 (truncated) # float x = 3.456 # strings — both quotes valid s1 = "abc" # same as 'abc' s2 = 'abc' s3 = "matt's" # unmatched within string OK

⬜ Whitespace Significance

Key Rule: Python uses indentation to define code blocks instead of curly braces {}. Indentation is MANDATORY, not optional!

Rules of Whitespace

  • Use a newline to end a line of code
  • Use \ when you must continue to the next line
  • No {} braces — use consistent indentation
  • First line with less indentation is outside the block
  • First line with more indentation starts a nested block
  • Colons : signal the start of a new block
  • You may use tabs OR spaces — but must be consistent
  • In Java indenting is optional; in Python it is required

Code Block Example

# hello3.py def hello(): print("Hello, world!") # ← indented (inside) print("How are you?") # ← indented (inside) # main — calls hello twice (not indented = outside) hello() hello()

Python vs Java — Block Style Comparison

✅ Python (indentation)

if x > 0: print("positive") print("done")

Java (curly braces)

if (x > 0) { System.out.println("pos"); System.out.println("done"); }

🔗 Assignment

What is Binding?

  • Binding = setting a name to hold a reference to an object
  • Assignment creates references, not copies
  • Names do not have intrinsic types — objects have types
  • Python determines the type of the reference automatically
  • You create a name the first time it appears on the left side of assignment
  • Unreferenced objects deleted via garbage collection
  • Python uses reference semantics

Assignment Forms

# Simple assignment x = 3 # Multiple assignment at once x, y = 2, 3 print(x) # 2 print(y) # 3 # Swap values (elegant Python trick!) x, y = y, x # Chained assignment a = b = x = 2 # all equal 2

Reference vs Copy Concept

When you write y = x, you're creating another name tag pointing to the same object. No new object is created — it's a reference!

x = 5 # x → integer object 5 y = x # y → SAME integer object 5 (reference) x = 10 # x → new integer object 10 # y still → 5 (wasn't affected) print(y) # 5

📛 Naming Rules & Conventions

Naming Rules (MUST follow)

  • Names are case sensitive — Bob ≠ bob
  • Cannot start with a number
  • Can contain: letters, numbers, underscores
  • Valid: bob Bob _bob _2_bob_ bob_2 BoB
  • Cannot use reserved/keywords as names

Naming Conventions (Community standard)

  • joined_lower — for functions, methods, attributes
  • joined_lower or ALL_CAPS — for constants
  • StudlyCaps — for class names
  • camelCase — only to conform to pre-existing conventions
  • interface, _internal, __private — attribute visibility
